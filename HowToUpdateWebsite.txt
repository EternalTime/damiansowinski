HOW TO UPDATE YOUR WEBSITE
==========================

1. Edit your files locally in /Users/owlshome/Documents/MyWebPage

2. Open a terminal and navigate to your project:
   cd /Users/owlshome/Documents/MyWebPage

3. Preview your changes locally:
   bundle exec jekyll serve
   Then open http://localhost:4000/damiansowinski in your browser.
   Press Ctrl+C to stop the server when done.

4. Stage your changes:
   git add .

5. Commit with a descriptive message:
   git commit -m "Brief description of what you changed"

6. Push to GitHub:
   git push

GitHub Pages will automatically rebuild and publish your site at
https://damiansowinski.com within a minute or two.

That's it — you never need to touch Squarespace again.
Squarespace only holds the domain name and silently redirects
traffic to GitHub Pages.


NOTES
-----
- The _site/ folder is auto-generated by Jekyll and ignored by git.
  Never edit files in _site/ directly.
- If you change _config.yml, you must restart Jekyll (Ctrl+C, then
  bundle exec jekyll serve again) to see the changes locally.
- Font files are in assets/fonts/
- Images are in assets/images/
- PDFs (papers, CV) are in assets/docs/
- Press photos are in assets/images/press/
